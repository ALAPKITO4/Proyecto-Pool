# ⚡ Quick Reference - Sistema de Estados\n\n## 🎯 En Una Página\n\n---\n\n## El Problema\n\n```javascript\n// ❌ ANTES\nuser.acepta()  // Solo en localStorage, NO en Firestore\n// Estado siempre \"pendiente\"\n// Creador no ve cambios\n// Datos se pierden si se borra localStorage\n```\n\n## La Solución\n\n```javascript\n// ✅ AHORA\nuser.acepta()  // En Firestore + localStorage\n// Estado \"aceptado\" con timestamp\n// Creador ve cambios EN VIVO\n// Datos persistentes en Firebase\n```\n\n---\n\n## Estructura de Participantes\n\n```javascript\n{\n  nombre: \"Juan García\",\n  telefono: \"351 1234567\",\n  estado: \"aceptado\",              // ← NUEVO: \"pendiente\" | \"aceptado\" | \"rechazado\"\n  acceptedAt: \"2026-04-19T10:00Z\",  // ← NUEVO: Cuándo aceptó\n  rejectedAt: null                 // ← NUEVO: Cuándo rechazó (si rechazó)\n}\n```\n\n---\n\n## Flujo de Aceptación\n\n```\nUsuario B abre invitación\n       ↓\nsubscribeToPoolUpdates() activado ← Escucha cambios\n       ↓\nUsuario B click \"Aceptar\"\n       ↓\nacceptPoolInvitation() \n  └─ participantes[1].estado = \"aceptado\" ← Cambio local\n  └─ Firestore.update(...) ← GUARDA EN BD\n       ↓\nonSnapshot se dispara ← Escuchador activo\n       ↓\nUsuario A ve cambio automáticamente\n```\n\n---\n\n## Console Logs Esperados\n\n### Aceptar\n```\n✅ Guardado en Firestore - Participantes actualizados\n📊 Aceptados: 2/3, Pendientes: 1/3, Rechazados: 0/3\n```\n\n### Sincronización Real\n```\n🔄 Pool actualizado en tiempo real: Escuela\n📊 Participantes: Juan(aceptado), María(aceptado), Pedro(pendiente)\n```\n\n---\n\n## Funciones Clave\n\n| Función | Qué Hace | Nuevo? |\n|---------|----------|--------|\n| `confirmPool()` | Crear pool con `participantes` | Mejorada |\n| `acceptPoolInvitation()` | Aceptar y guardar en Firestore | Mejorada |\n| `rejectPoolInvitation()` | Rechazar y guardar | ✅ Nuevo |\n| `subscribeToPoolUpdates()` | onSnapshot listener | Mejorada |\n| `showParticipantsSummary()` | Resumen en Console | ✅ Nuevo |\n| `showPoolDetails()` | UI con badges de estado | Mejorada |\n\n---\n\n## Comandos de Test Rápido\n\n### Ver Pool Actual\n```javascript\npoolsEvents[0]\n```\n\n### Ver Participantes\n```javascript\npoolsEvents[0].participantes\n```\n\n### Ver Quién Aceptó\n```javascript\npoolsEvents[0].participantes.filter(p => p.estado === 'aceptado')\n```\n\n### Ver Quién Rechazó\n```javascript\npoolsEvents[0].participantes.filter(p => p.estado === 'rechazado')\n```\n\n### Mostrar Resumen\n```javascript\nshowParticipantsSummary(poolsEvents[0].id)\n```\n\n---\n\n## Verificar Firestore\n\n1.
